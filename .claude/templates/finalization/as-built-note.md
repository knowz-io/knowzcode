# As-Built Notes Template

Add to spec files during finalization when implementation deviated from original spec.

## Format

```markdown
## As-Built Notes

### {date} - Finalization

| Spec Requirement | Actual Implementation | Reason |
|------------------|----------------------|--------|
| {original} | {as-built} | {rationale} |
```

## Usage

1. Compare spec requirements with actual implementation
2. Document any deviations
3. Record rationale for changes
4. Append to relevant spec file
